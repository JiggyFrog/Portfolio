Macro Tracker

Since I am big into fitness I decided to make an application to track your macronutrients. This would be my first time using react native (excluding a really barebones tutorial) and I would say it came out really good especially for the first time ever creating a project using react native.

How I started

I started by learning how to actually use react-native for a week by creating a quick task tracker following a tutorial. After, I proceeded by designing how I would like my app to look in figma so that It would hopefully look good. Then, I started by creating an expo project so I could run it on IOS because I do not have an Android device. Once I was settled in it was time to start programming the essential functions!

Designing the main components

After the base layout was down and in code, I began to design api getting functions and started looking for an API to use to get the food from barcode. I found OpenFoodFacts.org and could get information from it using their API, I used the default expo barcode scanner from the camera in order to get the barcode of the food the user was scanning.

After I could get requests I would have to store them somewhere. Since I am not really about buying a database and this was more of a test project I decided that I would instead store it on the local device which isn't what I would do if it was going to go into production, however, I don't think I am anywhere close to releasing this publicly yet.

After they were stored I created a couple new widgets in order to display the foods that the person logged and also gave them the option to edit the open food facts database in case if the data for the food they logged wasn't available. With this the app was actually pretty fleshed out and this is where I stopped for now in order to pursue a new idea that I wanted to try, AI
